# 미연시 화면/연출 제작 지침

이 문서는 다음 인터랙티브 소설을 만들 때 화면 구성, 이미지, 글상자, 선택지, 버튼, 전환 효과를 일관되게 잡기 위한 기준이다. 목표는 개발자용 UI가 아니라, 독자가 바로 미연시 장면으로 받아들이는 화면이다.

## 핵심 원칙

- 첫 화면부터 실제 작품 체험이어야 한다. 설명 페이지나 개발자용 대시보드처럼 보이면 안 된다.
- 한 컷에 모든 정보를 넣지 않는다. 이미지와 글상자의 비율을 지키고, 긴 문장은 여러 컷으로 나눈다.
- 화면 정보는 한국어 독자 기준으로 배치한다. 내부 ID, 후보 상태, 영문 개발 라벨은 노출하지 않는다.
- 배경, 캐릭터, 글상자는 분리해서 관리한다. 이미지 안에 글상자를 합성하지 않는다.
- 모든 주요 화면은 데스크톱, 좁은 브라우저, 모바일 폭에서 글자 겹침과 버튼 잘림이 없어야 한다.

## 화면 레이어

기본 장면은 아래 순서로 쌓는다.

1. 배경 이미지
2. 어두운 스크림
3. 캐릭터 스프라이트
4. 장소 배지와 상단 버튼
5. 글상자
6. 선택지, 저장/불러오기, 회상, 생각해보기 같은 오버레이
7. 장면 전환 오버레이

배경은 장면의 정서를 먼저 전달해야 한다. 너무 흐리거나 어두워서 장소를 알아보기 어렵다면 실패다. 스크림은 글과 캐릭터가 읽히는 정도까지만 사용한다.

## 캐릭터 이미지

- 기본 스프라이트는 투명 배경 PNG로 준비한다.
- 권장 원본 크기는 `720x900`이다. 전신을 포함해도 되지만 화면에는 글상자 뒤에서 상반신만 보이게 배치한다.
- 전신 이미지를 화면 위에 통째로 띄우지 않는다. 캐릭터 바닥선을 글상자 뒤로 내려 하반신이 가려지게 한다.
- 같은 캐릭터는 컷마다 얼굴, 체형, 색, 비율이 달라지면 안 된다. 새 표정이 필요해도 동일 캐릭터로 보이는 일관성을 우선한다.
- 캐릭터가 오른쪽에 있으면 왼쪽을 보고, 왼쪽에 있으면 오른쪽을 보게 한다.
- 대화 중인 캐릭터만 밝게 보이고, 나머지는 약하게 dim 처리한다.
- 해설 컷의 캐릭터 밝기는 해설의 성격에 따라 정한다.
  - 객관적인 장소/상황 설명은 양쪽 캐릭터를 약하게 dim 처리해 배경과 상황에 시선이 가게 한다.
  - 두 인물의 관계, 라포, 감정 교류가 핵심인 해설은 대사가 아니어도 관련 인물을 밝게 보여준다.
  - 주인공의 내면에 가까운 해설이나 속생각은 주인공을 밝게, 상대 인물은 약하게 dim 처리한다.
- 캐릭터는 글상자와 접해 있어야 한다. 글상자 위에 따로 떠 있거나 바닥 없이 공중에 매달린 느낌을 피한다.

## 글상자

- 글상자는 하단에 고정된 미연시 대화창으로 둔다.
- 높이는 어느 정도 고정한다. 텍스트 양에 따라 글상자 높이가 크게 바뀌면 캐릭터 위치가 흔들린다.
- 글상자 내부 스크롤은 쓰지 않는다. 글상자 안에 보이지 않는 텍스트가 생기면 컷을 나눈다.
- 한 컷의 글은 보통 1-2문장으로 제한한다. 긴 해설은 여러 컷으로 분리한다.
- 화자 이름표가 없는 짧은 해설 컷은 더 낮은 글상자를 써도 된다. 다만 캐릭터 기준선은 흔들리지 않아야 한다.
- 회상 장면은 글상자 색과 배경 톤을 바꿔 현재 시점과 구분한다.
- 대사, 해설, 지문, 속생각은 시각 톤을 다르게 한다. 속생각은 괄호나 별도 톤으로 조용하게 보여준다.

## 글상자 안의 버튼

- `지난 기록`, `저장`, `불러오기`, `가-`, `가+`는 글상자 하단 왼쪽에 작게 둔다.
- `이전`, `다음` 또는 `선택`은 글상자 하단 오른쪽에 작게 둔다.
- 버튼은 보조 수단이다. 기본 진행은 화면 탭/클릭으로 가능해야 한다.
- 버튼 글씨와 크기는 대사보다 작아야 한다. 버튼이 이야기보다 시선을 많이 가져가면 안 된다.
- 저장은 `기본 저장 + 자동 저장 1` 구조를 기본으로 한다. 불필요하게 많은 슬롯 목록을 보여주지 않는다.

## 장소와 상단 UI

- 장소 배지는 화면 좌측 상단에 작게 둔다.
- 작품 제목, 원작명, 상태값 같은 무거운 정보는 플레이 화면 상단에 반복 노출하지 않는다.
- 우측 상단에는 `전체 화면`, `목록` 정도만 둔다.
- 장소 배지는 장면 이해를 돕는 정도로만 쓰고, `1/2`, `2/2` 같은 컷 번호는 노출하지 않는다.

## 선택지

- 선택지는 하단 글상자에 붙이지 말고 화면 중앙 팝업으로 띄운다.
- 선택지 팝업은 화면의 상하 가운데에 정렬한다.
- 선택 버튼은 세로 메뉴형으로 배치한다. 한 줄에 여러 선택지를 빽빽하게 넣지 않는다.
- 선택 문구는 짧고 문학적인 행동 문장으로 쓴다.
- 선택지에는 정답/오답 느낌을 주지 않는다.
- 선택 후에는 즉시 장면을 바꾸지 말고 짧은 전환을 넣는다.

선택지 팝업의 권장 구성:

- 작은 선택 제목
- 장식선 또는 얇은 테두리
- 세로 선택 버튼 2-3개
- 작은 `지난 기록` 버튼
- 배경 블러 또는 어두운 스크림

## 선택 이후 전환

선택 이후에는 미연시식 전환을 넣는다.

1. 선택 팝업이 사라진다.
2. 짧은 암전 오버레이가 뜬다.
3. `선택한 길`, 선택 문구, 다음 장소를 보여준다.
4. 새 배경으로 바뀐다.
5. 장소 배지, 캐릭터, 글상자가 부드럽게 나타난다.

전환 시간은 짧게 유지한다. 1초 안팎이면 충분하다. 연출이 이야기 진행을 방해하면 안 된다.

## 장면 연출

- 장면 시작 시 배경, 캐릭터, 글상자는 부드럽게 들어온다.
- 회상 진입은 흐림, 색 변화, 글상자 톤 변화로 표현한다.
- 컷씬 지시가 있는 경우에도 텍스트로 설명하지 말고 화면 전환, 캐릭터 등장/퇴장, dim 처리로 해결한다.
- 효과는 반복해서 과하게 쓰지 않는다. 중요한 전환, 회상, 선택 이후에만 강조한다.
- 모션 민감 사용자를 위해 `prefers-reduced-motion`에서는 애니메이션을 끌 수 있어야 한다.

## 첫 화면 스토리 선택

- 첫 화면 제목은 서비스명 또는 앱 이름 중심으로 둔다.
- 스토리 카드는 간결해야 한다. 긴 설명은 접기/펴기로 제공한다.
- 기본 카드에는 제목, 짧은 소개, 시작 버튼만 명확히 보이면 된다.
- `review-candidate`, `folktale-adaptation` 같은 개발자 중심 문구는 숨긴다.

## 생각해보기

- 결말 이후의 `생각해보기`는 전체 화면에 가까운 팝업으로 띄운다.
- 결말마다 다른 선택 회고와 질문을 제공한다.
- 질문은 사용자를 평가하지 않고, 선택의 무게를 돌아보게 해야 한다.

## 금지 사항

- 글상자 내부 스크롤
- 화면에 개발자용 상태값 노출
- 한 컷에 과도한 설명문 넣기
- 캐릭터 전신을 화면 위에 통째로 띄우기
- 컷마다 다른 캐릭터처럼 보이는 이미지 사용
- 선택지를 하단 작은 패널에 끼워 넣기
- 정답/오답처럼 보이는 선택지 표현
- 버튼과 텍스트가 겹치는 레이아웃

## 제작 체크리스트

- 배경, 캐릭터, 글상자 비율이 안정적인가?
- 캐릭터가 글상자 뒤에서 상반신 컷처럼 보이는가?
- 왼쪽 캐릭터는 오른쪽을, 오른쪽 캐릭터는 왼쪽을 보는가?
- 해설 컷에서 객관 설명, 관계 중심 설명, 속생각에 맞춰 캐릭터 밝기를 다르게 처리했는가?
- 글상자 안의 모든 텍스트가 스크롤 없이 읽히는가?
- 짧은 대사와 긴 해설의 글상자 높이가 너무 흔들리지 않는가?
- 선택지는 화면 중앙 팝업으로 뜨는가?
- 선택 이후 전환이 자연스럽게 들어가는가?
- 저장/불러오기/지난 기록/글자 크기 버튼이 작고 보조적으로 보이는가?
- 모바일 폭에서 버튼과 텍스트가 겹치지 않는가?
- 브라우저에서 실제 장면, 선택지, 결말, 생각해보기를 확인했는가?
